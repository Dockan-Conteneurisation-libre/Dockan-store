Dockan Store

A local catalog of self-hosted app templates for Dockan. Install an app, let the Store import its prebuilt images, then run it with `dockan compose up`.

git clone https://github.com/Dockan-Conteneurisation-libre/Dockan-Store
cd Dockan-Store
./dockan-store list
./dockan-store install wordpress
cd ~/dockan-apps/wordpress
dockan compose up

Apps

Templates include compose files, persistent volumes, ports, healthchecks, prebuilt image names, and install notes. Search by name, category, or image.

Images

Each app card lists the exact Dockan images it needs. Releases include a generated image registry with `registry/index.tsv` plus prebuilt archives in `registry/images/`, so installs can import images without asking the user to build them. The release also publishes one smaller image pack per app.

./dockan-store images APP_ID
./dockan-store images all
./dockan-store build-images all
./dockan-store build-images mysql:local
DOCKAN_STORE_ENGINE=docker ./dockan-store build-images mysql:local
./dockan-store export-app-images wordpress

# release assets:
registry/index.tsv
registry/images/*.tar.gz
registry/images/*.tar.gz.sha256
dockan-store-images-APP_ID.tar.gz

How Install Works

Dockan Store imports the required prebuilt Dockan images from its bundled registry, then copies the template into your app folder. The app developer does not build images manually.

./dockan-store install APP_ID
cd ~/dockan-apps/APP_ID
dockan compose up
dockan compose health

# custom target:
./dockan-store install APP_ID /srv/apps/APP_ID

# prepare images only:
./dockan-store images APP_ID
